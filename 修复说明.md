**文件**: `frontend/src/components/SongPoemSection.vue`

**问题**: DeepSeek 偶尔返回空响应，导致前端无法显示内容

## 🎯 核心改进

### 音频可视化逻辑
```javascript
// 旧版本：立即初始化（会导致播放失败）
onMounted(() => {
  initAudioVisualizer()  // ❌ 立即连接audio元素
})

// 新版本：延迟初始化（不影响播放）
onMounted(() => {
  draw()  // ✅ 先开始绘制
})

watch(() => props.isPlaying, (newVal) => {
  if (newVal && !isInitialized) {
    setTimeout(() => {
      initAudioVisualizer()  // ✅ 播放时才初始化
    }, 500)
  }
})
```

### 错误处理
```javascript
// 避免重复连接
try {
  sourceNode = audioContext.createMediaElementSource(audioElement)
  sourceNode.connect(analyser)
  analyser.connect(audioContext.destination)
} catch (error) {
  if (error.name === 'InvalidStateError') {
    console.log('Audio element already connected, skipping')
  }
}
```

## 🐛 如果还有问题

### 问题1: 音乐无法播放
**检查**:
1. 打开浏览器控制台（F12）
2. 查看是否有错误信息
3. 检查 Network 标签，看音频URL是否正确

**可能原因**:
- 网易云音乐 Cookie 过期
- 音频URL无效
- CORS问题

### 问题2: DeepSeek 返回空响应
**检查**:
1. 查看后端日志
2. 确认 DEEPSEEK_API_KEY 是否正确
3. 检查网络连接

**临时方案**:
- 系统会自动返回友好的默认消息
- 可以直接说"播放周杰伦的晴天"

### 问题3: 音频可视化不工作
**检查**:
1. 浏览器是否支持 Web Audio API
2. 控制台是否有错误
3. 音乐是否正在播放

**说明**:
- 即使可视化失败，音乐播放不受影响
- 会显示简单的动画效果

## 💡 技术要点

### Web Audio API 注意事项
- `createMediaElementSource()` 只能调用一次
- 必须在用户交互后才能创建 AudioContext
- 连接后音频会通过 AudioContext 播放

### DeepSeek API
- 使用 JSON 模式返回结构化数据
- 需要完善的错误处理
- 空响应需要友好的降级方案

### Vue 3 响应式
- 使用 watch 监听播放状态
- 延迟初始化避免竞态条件
- 清理资源避免内存泄漏
