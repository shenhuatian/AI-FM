## 🎯 优化目标
实现混合推荐策略（70% 探索新歌 + 30% 舒适区），让 AI DJ 能够推荐更多样化的音乐，不再局限于用户音乐库。

## ✅ 已完成的功能

### 1. 网易云 API 扩展 (`src/music/ncm.js`)

新增了以下 API 方法：

#### 1.1 相似歌曲推荐
```javascript
await ncm.getSimilarSongs(songId, limit)
```
- 基于一首歌曲推荐相似风格的歌曲
- 使用网易云的 `/simi/song` 接口
- 用于自动补充推荐列表

#### 1.2 相似艺术家推荐
```javascript
await ncm.getSimilarArtists(artistId, limit)
```
- 基于一位艺术家推荐相似风格的艺术家
- 使用网易云的 `/simi/artist` 接口
- 用于发现新的艺术家

#### 1.3 艺术家热门歌曲
```javascript
await ncm.getArtistTopSongs(artistId, limit)
```
- 获取艺术家的热门歌曲（最多 50 首）
- 包含热度信息（`popularity` 字段）
- 支持热度筛选

#### 1.4 艺术家搜索
```javascript
await ncm.searchArtist(keyword, limit)
```
- 搜索艺术家（使用 `type=100`）
- 返回艺术家 ID、名称、头像等信息

### 2. 推荐策略模块 (`src/brain/recommendation-strategy.js`)

全新的推荐策略系统，实现智能混合推荐。

#### 2.1 混合推荐
```javascript
await strategy.hybridRecommend(query, totalCount, options)
```

**核心逻辑：**
- 70% 探索模式：从网易云搜索新歌
- 30% 舒适区模式：从用户音乐库推荐

**参数：**
- `query`: 用户查询
- `totalCount`: 推荐歌曲总数
- `options`:
  - `artistName`: 指定艺术家
  - `mood`: 心情标签
  - `preferFresh`: 是否偏好冷门歌曲
  - `excludeRecent`: 是否排除最近播放

**推荐流程：**
```
用户请求 → 分配比例（70% 探索 + 30% 舒适区）
              ↓
    ┌─────────┴─────────┐
    ↓                   ↓
探索模式            舒适区模式
    ↓                   ↓
网易云搜索          音乐库搜索
    ↓                   ↓
相似推荐            语义搜索
    ↓                   ↓
    └─────────┬─────────┘
              ↓
        去重 + 合并
              ↓
        返回结果
```

#### 2.2 智能推荐
```javascript
await strategy.smartRecommend(userMessage, count)
```

**功能：**
- 自动分析用户意图
- 识别艺术家、心情、冷门偏好
- 自动调用混合推荐

**意图识别：**
- **艺术家识别**：
  - "陶喆的歌" → `artist: "陶喆"`
  - "听周杰伦的" → `artist: "周杰伦"`
  
- **冷门偏好识别**：
  - "没怎么听过" → `preferFresh: true`
  - "小众"、"冷门"、"深度" → `preferFresh: true`
  
- **心情识别**：
  - "开心"、"快乐" → `mood: "开心"`
  - "悲伤"、"难过" → `mood: "悲伤"`
  - "放松"、"舒缓" → `mood: "放松"`

#### 2.3 热度筛选
```javascript
strategy.filterByPopularity(songs, level)
```

**功能：**
- 根据歌曲热度筛选
- 支持三个级别：
  - `low`: 冷门歌曲（热度 < 平均值）
  - `medium`: 全部歌曲
  - `high`: 热门歌曲（热度 >= 平均值）

**应用场景：**
- 用户说"没怎么听过" → 筛选冷门歌曲
- 用户说"热门歌曲" → 筛选热门歌曲

### 3. 主流程集成 (`src/index.js`)

#### 3.1 自动补充推荐

**触发条件：**
- AI 推荐了 N 首歌
- 实际只找到 M 首（M < N）

**补充逻辑：**
1. 使用第一首成功找到的歌曲作为种子
2. 调用 `getSimilarSongs()` 获取相似歌曲
3. 过滤重复和最近播放
4. 补充到目标数量

**示例：**
```
AI 推荐: 5 首歌
实际找到: 2 首
↓
自动补充: 3 首相似歌曲
↓
最终返回: 5 首歌
```

#### 3.2 去重优化

**去重策略：**
- 按歌曲名称去重（不区分大小写）
- 避免同一首歌的不同版本重复推荐
- 例如："告白气球 - 周杰伦" 和 "告白气球 - 周杰伦/方文山" 只保留一首

## 🎮 使用示例

### 场景 1：用户说"我想听 Kanye 的 Never See Me Again"

**之前的行为：**
- 只在音乐库搜索
- 找不到 → 推荐音乐库中的其他歌

**现在的行为：**
1. 先在音乐库搜索（30% 概率）
2. 如果找不到，调用网易云搜索
3. 同时推荐相似风格的歌曲

### 场景 2：用户说"推荐一些说唱歌曲"

**推荐策略：**
1. 从音乐库找 2 首说唱（30%）
2. 从网易云搜索 5 首新的说唱（70%）
   - 使用关键词搜索："说唱 hip-hop"
   - 筛选条件：热度适中（避免太热门）
   - 排除用户最近听过的

### 场景 3：用户说"来点陶喆的歌，我想要听一些我没怎么听过的"

**推荐策略：**
1. 识别艺术家：陶喆
2. 识别偏好：冷门歌曲（`preferFresh: true`）
3. 从音乐库找 1-2 首陶喆（30%）
4. 从网易云搜索陶喆的其他歌（70%）
   - 使用 `getArtistTopSongs()` 获取所有歌曲
   - 使用 `filterByPopularity(songs, 'low')` 筛选冷门曲目
   - 排除最近播放

### 场景 4：AI 推荐的歌曲找不到

**自动补充：**
1. AI 推荐了 5 首歌
2. 只找到 2 首
3. 系统自动使用第一首歌作为种子
4. 调用 `getSimilarSongs()` 补充 3 首相似歌曲
5. 最终返回 5 首歌
